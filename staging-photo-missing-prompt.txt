Issue Summary (Staging/Prod)

Symptoms:
- Backend/API is healthy and product import runs.
- Product endpoints return data, but many image requests under /media/products/*.jpg|png return 404.
- Browser previously showed mixed-content warnings for image URLs (http links on https page).

Observed Logs:
- Import startup reports:
  - 1812 products processed
  - 1705 images found
  - indicates ~107 products have no matching source image
- Django warns:
  - Not Found: /media/products/<file>
- Migration warning still present:
  - models in orders/users changed but no migration files applied.

Likely Root Causes:
1) Partial dataset image coverage
- Import source does not contain matching image files for every product reference.

2) Stale image references in DB
- Historical imports left some image paths that no longer exist on disk.

3) Serialization/proxy URL shape
- Absolute image URLs with scheme mismatch can trigger mixed-content behavior behind reverse proxy.

4) Media serving/routing had to be explicitly enabled in prod
- /media path must route to backend and Django must serve media URL in this deployment shape.

Fixes already implemented:
- Traefik backend routing includes /media.
- Django URL config serves MEDIA_URL.
- Import startup fallback no longer crashes if master CSV missing.
- Product image serializer normalizes to relative paths and now returns null when file is missing.
- Import update clears stale image references when file is absent.

Expected behavior after redeploy:
- Missing-file products return image=null and frontend shows placeholders.
- Existing media files load from /media/products/... without mixed-content issues.

Remaining actions:
1) Redeploy backend with latest image and compose labels.
2) Run import cycle once (startup import already enabled).
3) Verify API payloads return image=null for products with missing files.
4) (Separate) generate/apply missing migrations for orders/users to remove schema drift warnings.

Prompt for further debugging (if needed):
"Investigate why specific product references still resolve to missing files under /media/products in staging. Compare DB image field values vs actual files in MEDIA_ROOT/products, and produce a report of mismatches with counts by extension and reference. Then propose either deterministic filename normalization in import_product_data or a backfill command to repair image paths."
